# 앱 홍보 자동화 운영 가이드

> 목표: 각 플랫폼의 정책을 지키면서 반복 가능한 홍보 시스템 구축.

## 공통 원칙

-   동일한 글을 여러 플랫폼에 복붙하지 않는다.
-   하루 게시 빈도를 천천히 늘린다.
-   AI가 초안을 만들고 최종 검토 후 게시한다.
-   댓글에는 성실하게 답변한다.
-   허위 리뷰, 가짜 계정, 리뷰 구매를 하지 않는다.

## Google Play

-   업데이트마다 변경사항 작성.
-   스크린샷/A-B 테스트.
-   핵심 키워드 자연스럽게 포함.

## Reddit

-   계정을 먼저 활동시킨다.
-   관련 서브레딧 규칙을 읽는다.
-   개발기, 피드백 요청 중심.
-   같은 글 대량 게시 금지.

## X

-   짧고 영상 중심.
-   개발 과정 공유.
-   하루 1\~3회 정도.
-   같은 문구 반복 금지.

## Threads

-   개발 비하인드, 질문형 글.
-   댓글 적극 참여.

## YouTube Shorts

-   15\~30초.
-   첫 3초에 핵심 장면.
-   업데이트마다 새 영상.

## TikTok

-   게임 플레이 중심.
-   트렌드 사운드는 저작권 확인.
-   동일 영상 반복 업로드 지양.

## Discord

-   자신의 서버 운영.
-   공지보다 대화 비율 높게.

## 블로그

-   검색 키워드 중심.
-   기능 소개보다 문제 해결형 글 작성.

## 자동화 파이프라인

1.  GitHub Release 감지
2.  GPT로 플랫폼별 글 생성
3.  이미지/영상 첨부
4.  예약 발행
5.  반응 분석
6.  다음 글 개선

## 밴을 피하기 위한 체크리스트

-   스팸성 반복 게시 금지
-   자동 댓글 금지
-   과도한 DM 금지
-   플랫폼별 API 및 이용약관 준수
-   AI 생성 글도 플랫폼별 형식에 맞게 수정
